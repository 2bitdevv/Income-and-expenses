# บันทึกรายรับ-รายจ่าย

เว็บแอปสำหรับบันทึกรายรับรายจ่ายแบบหน้าเดียว ใช้งานร่วมกับ Supabase สำหรับเก็บข้อมูล และสามารถส่งออกรายงาน PDF ได้

เว็บไซต์ที่ deploy แล้ว:
[https://income-and-expenses-delta.vercel.app/](https://income-and-expenses-delta.vercel.app/)

## ฟีเจอร์หลัก

- เพิ่มรายการรายรับและรายจ่าย
- แก้ไขและลบรายการเดิม
- สรุปยอดรายรับ รายจ่าย และคงเหลือแบบอัตโนมัติ
- ค้นหาและกรองรายการตามประเภทหรือเดือน
- ส่งออกรายงาน PDF
- ใช้งานภาษาไทยและฟอนต์ `Noto Sans Thai`
- เชื่อมต่อฐานข้อมูลผ่าน Supabase

## โครงสร้างไฟล์หลัก

- `index.html` หน้าเว็บหลัก พร้อม CSS และ JavaScript
- `config.js` ไฟล์ตั้งค่า Supabase
- `package.json` dependency ที่ใช้ในโปรเจกต์

## การตั้งค่า Supabase

แก้ไฟล์ `config.js` ให้เป็นค่าโปรเจกต์จริงของคุณ:

```js
window.APP_CONFIG = {
  SUPABASE_URL: "https://YOUR_PROJECT_ID.supabase.co",
  SUPABASE_ANON_KEY: "YOUR_SUPABASE_ANON_KEY"
};
```

ข้อสำคัญ:

- ใช้ `anon key` เท่านั้น
- ห้ามใช้ `service_role key` บนฝั่ง frontend
- โปรเจกต์ Supabase ต้องมี table ชื่อ `budget_entries`

## โครงสร้างตารางที่ต้องมี

ตัวอย่าง SQL สำหรับสร้างตาราง:

```sql
create table if not exists public.budget_entries (
  id bigint generated by default as identity primary key,
  type text not null check (type in ('income', 'expense')),
  title text not null,
  category text,
  amount numeric(12,2) not null default 0,
  date date not null,
  note text,
  created_at timestamptz not null default now()
);
```

## ตัวอย่าง Policy เบื้องต้น

ถ้าเปิดใช้งาน RLS ต้องมี policy ที่อนุญาตอย่างน้อย `select`, `insert`, `update`, `delete`

ตัวอย่างสำหรับทดสอบแบบง่าย:

```sql
alter table public.budget_entries enable row level security;

create policy "Allow public read"
on public.budget_entries
for select
to anon
using (true);

create policy "Allow public insert"
on public.budget_entries
for insert
to anon
with check (true);

create policy "Allow public update"
on public.budget_entries
for update
to anon
using (true)
with check (true);

create policy "Allow public delete"
on public.budget_entries
for delete
to anon
using (true);
```

หมายเหตุ:
ถ้าใช้งานจริง แนะนำให้ปรับ policy ให้ปลอดภัยกว่านี้ตามระบบผู้ใช้ของคุณ

## การรันในเครื่อง

โปรเจกต์นี้เป็น static site จึงเปิดใช้งานได้หลายแบบ

### วิธีง่ายสุด

เปิดไฟล์ `index.html` ตรง ๆ ในเบราว์เซอร์

### ใช้ local server

ถ้ามี VS Code สามารถใช้ Live Server ได้ หรือใช้คำสั่ง:

```bash
npx serve .
```

จากนั้นเปิดเบราว์เซอร์ไปที่ URL ที่เครื่องแสดง

## Deploy ขึ้น Vercel

### วิธีผ่าน GitHub

1. push โปรเจกต์ขึ้น GitHub
2. เข้า [https://vercel.com/new](https://vercel.com/new)
3. เลือก repository นี้
4. ตั้งค่าเป็น:
   - Framework Preset: `Other`
   - Build Command: เว้นว่าง
   - Output Directory: เว้นว่าง
5. กด Deploy

### วิธีผ่าน Vercel CLI

```bash
npm i -g vercel
vercel
```

แล้วเลือกโฟลเดอร์โปรเจกต์นี้ตามขั้นตอน

## การใช้งาน

1. เลือกประเภท `รายรับ` หรือ `รายจ่าย`
2. กรอกจำนวนเงิน รายละเอียด หมวดหมู่ วันที่ และหมายเหตุ
3. กด `บันทึกรายการ`
4. ดูรายการทั้งหมดในแท็บ `รายการทั้งหมด`
5. ใช้ตัวกรองเพื่อค้นหาข้อมูล
6. กด `ดาวน์โหลด PDF` เพื่อสร้างรายงาน

## ปัญหาที่พบบ่อย

### บันทึกไม่ได้

ตรวจสอบสิ่งต่อไปนี้:

- `config.js` ใส่ `SUPABASE_URL` ถูกต้อง
- `config.js` ใส่ `SUPABASE_ANON_KEY` ถูกต้อง
- ตาราง `budget_entries` มีอยู่จริง
- RLS policy อนุญาต `insert`

### โหลดข้อมูลไม่ได้

ตรวจสอบว่า:

- internet ใช้งานได้
- Supabase project ยัง online
- key และ URL มาจาก project เดียวกัน

### Deploy แล้วเว็บขึ้นแต่ข้อมูลไม่ทำงาน

ให้เช็กว่าไฟล์ `config.js` ถูกอัปขึ้นไปด้วย และค่าข้างในเป็นค่าจริง ไม่ใช่ placeholder
